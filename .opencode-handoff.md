## Estado — Feature: Página de Contacto ✅

### Fase 1 — Backend: API de contacto ✅
- Tabla `contact_messages` con columnas: `id`, `name`, `email`, `phone`, `area`, `subject`, `message`, `created_at`
- `POST /api/contact` — crea mensaje con validaciones inline (nombre >= 2 chars, email regex, asunto >= 3 chars, mensaje >= 10 y <= 2000 chars)
- Áreas de contacto: `ventas`, `soporte`, `reclamos`, `prensa`, `otros`
- Transporter mock (consola) + real (nodemailer SMTP con variables de entorno)
- 7 tests backend en `server/__tests__/routes/contact.test.ts`

### Fase 2 — Servicio + Hook de contacto ✅
- Tipos `ContactFormData`, `ContactResponse`, `ContactAreaInfo`, `ContactArea` en `src/types/contact.ts`
- `sendContactMessage()` en `src/services/contactService.ts` — POST a `/api/contact` con manejo de errores HTTP y de red
- `getContactAreas()` — devuelve lista de áreas
- `useContact()` hook en `src/hooks/useContact.ts` — estado loading/success/error/messageId, submit() y reset()
- 14 tests en `src/__tests__/contact.test.js` (servicio + hook)

### Fase 3 — UI de Contacto ✅
- `ContactPage.tsx` — página completa con formulario, validación inline, pantalla de éxito/error, feedback con useToast
- 6 campos: Nombre, Email, Teléfono, Área (select), Asunto, Mensaje (textarea con contador /2000)
- Diseño responsive mobile-first con grid sm:grid-cols-2, info de contacto en bg-gray-50
- Botones: "Enviar mensaje" (con loading state), "Cancelar", y en éxito: "Volver al inicio" + "Enviar otro mensaje"
- 20 tests en `src/__tests__/ContactPage.test.tsx` (componente mockeado)

### Fase 4 — Routing, integración y tests finales ✅
- Ruta `/contact` en Router.tsx — renderiza ContactPage directamente
- Botón "Contacto" en Header.tsx navega a `/contact` via `navigate('/contact')`
- Tests de navegación en `navigation.test.jsx` (clic en Contacto → /contact)
- 5 tests E2E en `e2e/contact.spec.js` (navegación, formulario válido, validación vacío, email inválido, header navigation)
- Fixes a tests E2E pre-existentes: URL patterns, selectores de precio/talle, navegación por imagen

### Fase 5 — Infraestructura transversal de email ✅
- `emailService.ts` — Servicio central con `sendEmail`, `sendWelcomeEmail`, `sendOrderConfirmationEmail` (con PDF adjunto), `sendContactEmail`. Modo mock si no hay SMTP configurado, modo real con nodemailer.
- `emailTemplates.ts` — Plantillas HTML responsivas inline: bienvenida (logo + catálogo), confirmación de compra (tabla items + total), contacto (datos remitente + mensaje para el área)
- `pdfService.ts` — Generación de PDF con pdfkit: factura con datos cliente, tabla productos, total, mensaje agradecimiento
- Integrado en `auth.ts` (welcome email tras registro), `orders.ts` (confirmación con PDF tras compra), `contact.ts` (HTML templates vía emailService)
- 19 tests en `server/__tests__/services/emailService.test.ts`
- **Requerido para modo real**: `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM` en entorno
